AKOM — Disaster Coordination Center. Live sensor feeds, vehicle fleets, camera networks, satellite tracking, maritime-air traffic, and OSINT events on an interactive 3D map. A working prototype built for the high-probability Marmara earthquake scenario.

Try the Cockpit Live ↗ Explore ↓

İstanbul faces a Mw 7+ Marmara earthquake within the next 30 years. AKOM is a browser-based cockpit built from scratch for command-and-control, situational awareness, and resource coordination in the first hours of a crisis.

// 01 · GLOBAL VIEW

KÜRE — Situational Awareness with 15,000+ Satellites

NASA BlueMarble basemap on MapLibre globe projection. Over 15,000 active satellites are computed in real time with the SGP4 orbital model inside a Web Worker and displayed live on the globe. Full-spectrum space domain awareness with operational overlay layers.

// 02 · COMMAND & CONTROL

C2 — Command & Control Kanban Board

Maven-style mission cards flow through a six-stage dispatch pipeline: DELIBERATE → DYNAMIC → PENDING → PAIRED → EXECUTION → COMPLETE. Each card has an attached detail drawer with unit pairing, approval chain, intelligence notes, and map fly-to.

// 03 · SURVEILLANCE

GÖRÜ — 632-Camera İBB Network + Viewshed Cones

Viewshed cones for 632 İBB PTZ cameras on the map, paired with coverage / blind-spot / redundancy analysis. The İstanbul tourism camera grid (18 sites) shows authentic recorded sample frames and short clips — honestly labeled SIM previews since İBB CCTV cannot be embedded in a browser.

// 03b · CAMERA MEDIA

Recorded Camera Samples

İBB's live camera streams cannot be shown in a browser (X-Frame-Options: SAMEORIGIN + CSP restriction). AKOM embraces this honesty: every camera tile displays server-side pre-captured real İstanbul imagery, labeled SIM. Live streaming can be attempted inside the Tauri desktop shell.

// 04 · OPERATIONS

OPERASYON — 3,000 Units, A* on Real Road Graph

Up to 3,000 simulated vehicle units move across İBB's real CKAN road data using the A* (A-star) pathfinding algorithm. Full-scale operations simulation with dispatch waves, priority routes, and district-level density heatmaps.

// 05 · SCENARIO ENGINE

Deterministic Mw 7.5 Marmara Scenario

Repeatable earthquake simulation: district-level deterministic damage attenuation based on site amplification, damage choropleth map, surge demand multiplier, dispatch waves, and infrastructure cascade. T+ stopwatch, PHASE indicator, and 1×/4×/16× speed control — ideal for operator training and planning.

// 06 · DUAL THEME

Dual Theme — Clean Command & Tactical Dark

AKOM offers two complementary themes: a clean command theme on white (daytime operations, reporting) and an amber-black tactical theme (night shift, crisis moment). Both themes work fully across all 10 mission tabs, with persistent preference storage.

// 07 · EVENT LEGEND

Map Event Legend — Color-Coded Emergency Types

C2 and OSINT event types are encoded with a consistent color palette: Collapse / MASCAL red, fire orange, flood blue, gas leak yellow, HAZMAT violet. Damage levels shown via gradient. Instantly recognizable in both themes.

// 08 · LIVE AIR TRAFFIC

Live ADS-B — Flights Over İstanbul via OpenSky

Commercial flights in İstanbul airspace are fetched in real time from the OpenSky Network public API and displayed on the map. Falls back to an honest SIM backup when offline — every data point is source-labeled (LIVE / SIM).

// 09 · HONESTY DOCTRINE

Every Value Source-Labeled — The Honesty Doctrine

AKOM's core principle: the operator must always know what is real and what is simulated. Every data point is labeled with its source — KRDAE · AFAD · İBB · ICAO · AIS · TLE · OSINT · EST · SIM. When cameras cannot be embedded in a browser, recorded frames are shown with a SIM label. This is not a weakness — it is a selling point.

// 10 · INFRASTRUCTURE CASCADE

Infrastructure Cascade Panel

Interactive panel modeling the progressive collapse of electricity, water, and natural gas networks post-earthquake. Fault propagation on the T+ timeline, district-level impact scores, and status tracking for critical infrastructure assets.

// ARCHITECTURE & METHODS

ARCHITECTURE & METHODS

An academic-level breakdown of the technical foundation. How do the pieces connect?

// 01 · RENDER PIPELINE

Single Shared WebGL2 Context

MapLibre GL v5 draws the basemap (vector tiles, WebGL2). deck.gl v9 draws the data overlays (3D) in the same context. Critical architectural decision: deck.gl is mounted as a middleware layer sharing MapLibre's WebGL context — not a second stacked canvas. This ensures elevated buildings, patrol routes, events, and satellites composite with depth-correct blending against the basemap in a single render loop.

Result: no Three.js — deck.gl is the 3D engine living inside the map's GL context. Layer types used: PolygonLayer (elevated buildings), PathLayer (patrol routes), ScatterplotLayer (events / OSINT / satellites / cameras), IconLayer + SimpleMeshLayer (MIL-STD symbols + 3D camera frustums), BitmapLayer (viewshed), HeatmapLayer (camera coverage density).

// 02 · STATE & REACTIVITY

Single Zustand Store, Selector Subscriptions

A single Zustand v5 store, partitioned into slices (entities / c2 / dock). Components subscribe to narrow selectors — a vehicle position update re-renders only the component that reads it.

subscribeWithSelector allows imperative consumers (map fly-to, simulation driver) to react to specific keys without React re-renders. This is how 3,000 moving units stay smooth: deck.gl layers read a single snapshot array, not per-unit React state.

// 03 · VEHICLE ROUTING

Real Road Graph + A*

İBB CKAN open data (tier-1 emergency roads + main arteries), compiled into an adjacency graph (road-graph.json) by scripts/build-road-graph.mjs: nodes = intersections, edges = segments weighted by haversine distance. Fragments are merged via a union-find proximity pass to ensure network connectivity.

PatrolEngine runs A* (haversine admissible heuristic) on this graph to route units on real İstanbul streets — no fake beat-loops. Performance strategy: lazy leg evaluation + per-tick route budget (MAX_ROUTES_PER_TICK); units park and queue, consuming a few A* solutions per snapshot. This keeps first-load and steady state within frame budget for 3,000 units.

// 04 · SATELLITES

SGP4 — Orbital Propagation in a Web Worker

Real TLE element sets from Celestrak (15,000+ objects). satellite.js SGP4 propagation runs off the main thread inside a dedicated Web Worker at ~1.5 Hz. Positions are returned as packed typed arrays (transferable) and decoded into a ScatterplotLayer on the globe. Category filtering is done via a bitmask pushed to the worker — the main thread never loops over 15k objects.

// 05 · SCENARIO ENGINE

Deterministic, Seeded, Repeatable

A pure ScenarioEngine: ALL randomness is consumed at construction time (seeded hash → district-level site amplification; infrastructure picks from real utility node IDs). Therefore tick(simMs) is a pure threshold-exceedance function — same seed ⇒ same run, independent of frame rate.

Outputs: ground motion attenuation × district-level site → damage choropleth (green→amber→red); a casualty/event surge demand multiplier (≈1→8→3 across phases); dispatch waves into the patrol engine; an infrastructure cascade. A 500 ms driver maps these into live cockpit state (KRDAE quake event, flash comms, MASCAL, OSINT surge, disabled utility nodes).

// 06 · LIVE DATA + SYNTHETIC NEST

Honesty Doctrine — A METHOD, not a flaw

Real feeds: OpenSky ADS-B (60 s poll, honest SIM fallback after repeated failures — production-live), AFAD alerts, Celestrak TLEs, İBB CKAN roads.

Simulated feeds: patrols, ELT distress beacons, AIS maritime, OSINT events, MOBESE camera previews, tipline/event intake.

There is no backend. Everything is either a client-side-fetched public API or generated in-app. Every value carries a source label (KRDAE / AFAD / İBB / ICAO / AIS / TLE / OSINT / EST / SIM). Source honesty is a deliberate methodological stance.

// 07 · CAMERA MEDIA

Server-Side Acquisition for Browser-Blocked Source

İBB CCTV cannot be embedded in a browser (X-Frame-Options + CSP + CORS). Method: server-side ffmpeg pass-through captures real still frames and short clips, served as static recorded samples, honestly labeled (RECORDED / REPRESENTATIVE). A working HLS server had to be discovered (kamerayayin.ibb.istanbul).

// 08 · BUILD, TYPES, SECURITY, DELIVERY

Zero-Warning Type-Safe Static SPA

React 19 + Vite 6 (SWC). TypeScript strict (noUncheckedIndexedAccess, noUnusedLocals) + ESLint 0-warning threshold. CSP-locked (single meta tag governs connect-src; script-src 'self' prohibits inline scripts).

Deployed as a pure static SPA to GitHub Pages at /akom-cockpit-live/ base. Optional Tauri v2 desktop shell bypasses browser CORS for live HLS.

Tested with Vitest (unit, ~200 tests) + Playwright (e2e); a micro benchmark measures patrol snapshot cost at 300 / 1,000 / 3,000 units.
